Este trabajo es sobre el enunciado del tp3 


### enunciaod del trabajo 

- ejercicio uno :Desarrollar un sitio web estático que simule un Sistema de Gestión de Stock
para un comercio multirubro (kiosco, librería y regalería). El sitio deberá
permitir visualizar productos, gestionar inventario y registrar transacciones,
aplicando los conceptos de HTML y CSS.

### los requisitos

-El sitio web debe cumplir con los siguientes puntos:

1. Estructura HTML y CSS
● Uso correcto de etiquetas semánticas (<header>, <section>,
<footer>, etc.).
● Hoja de estilo externa en CSS (styles.css) con:
○ Selectores de clase y/o ID.
○ Propiedades de color (hexadecimal, RGB, HSL), márgenes,
padding y bordes.
○ Estilos para tablas y formularios.

2. Contenido y Funcionalidad
● Página principal (index.html):
○ Presentación del comercio (nombre, rubros, logo).
○ Tabla de productos destacados (ejemplo: 5 productos por rubro
con columnas: Nombre, Precio, Stock).
○ Enlace a otras páginas (gestion.html, ventas.html).
● Página de gestión (gestion.html):
○ Formulario para agregar/editar productos (campos: Nombre,
Rubro, Precio, Stock).
○ Botón simulado "Guardar" (no requiere funcionalidad JS).
● Página de ventas (ventas.html):
○ Tabla de transacciones (columnas: Fecha, Producto, Cantidad,
Total).
○ Sección de "Pagos a cuenta" con lista ficticia.

3. Elementos Obligatorios
● CSS:
○ Uso de flexbox o grid para organizar al menos una sección.
○ Estilos para hover en botones/enlaces.
○ Comentarios en el código CSS explicando propiedades clave.
● HTML:
○ Imágenes locales (ejemplo: logos de rubros).
○ Atributos alt, title y style en al menos dos elementos.
○ Comentarios HTML descriptivos.

4. Publicación
● Repositorio GitHub con nombre: TP3-stock-[nombre-comercio]
(ejemplo: TP3-stock-multirubro).
● Publicación en GitHub Pages (Settings > Pages > rama main).
